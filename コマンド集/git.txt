ローカル優先でgithub強制更新
git status         # 変更がすべて commit 済みか
git log --oneline -5   # 直近コミットを把握
git branch backup-$(date +%Y%m%d-%H%M)
# 現在ブランチが main か確認
git symbolic-ref --short -q HEAD   # => main なら OK

# ローカル main を GitHub の main に強制プッシュ
git push --force-with-lease origin main







# 1. ローカルの変更をすべて破棄して、リモートの最新状態に強制的に合わせる
git fetch origin
git reset --hard origin/main  # ブランチ名がmainの場合。他のブランチなら適宜変更
git clean -fd



GitHub 側での公開鍵の登録手順
ブラウザで GitHub にログイン
右上のプロフィールアイコン → Settings
左のメニュー → SSH and GPG keys
New SSH key をクリック
Title: 任意（例: Xubuntu Dev Machine）
Key type: Authentication Key のまま
Key: 先ほどコピーした公開鍵の内容をペースト
Add SSH key を押す



接続テスト（Linux側）
ssh -T git@github.com

初回は fingerprint 確認が出ます：

The authenticity of host 'github.com (IPアドレス)' can't be established.
ED25519 key fingerprint is SHA256:xxxxxxxxxxxxxxxx.
Are you sure you want to continue connecting (yes/no/[fingerprint])?

→ yes と入力

成功時のメッセージ：
Hi shibazach! You've successfully authenticated, but GitHub does not provide shell access.
これが出ればOKです。


その後の運用

リポジトリのURLを SSH 用に統一する：

cd /path/to/your/repo
git remote set-url origin git@github.com:shibazach/<リポジトリ名>.git


以降は git push / git pull でパスワード入力不要。
